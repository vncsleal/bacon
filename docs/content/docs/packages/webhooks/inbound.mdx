bacon makes use of [Svix](https://svix.com) to handle inbound webhooks. Some of our packages take advantage of webhooks, for example:

- [Authentication](/migrations/authentication) — responds to user CRUD events.
- [Payments](/packages/payments) — handles Stripe events.
- [CMS](/packages/cms) — revalidates cached content.

## Securing your endpoints

To secure your webhook endpoints, Svix verifies that the webhook signature is valid by using a signing secret method. See the [Svix documentation](https://docs.svix.com/reception/verifying) for more details.

## Local Development

You can use Svix's [ingestion API](https://docs.svix.com/ingestion) or the [Svix CLI](https://docs.svix.com/cli) to forward events to your local server.

For Stripe webhooks, use the [Stripe CLI](https://stripe.com/docs/stripe-cli) to forward events:

```sh
stripe listen --forward-to localhost:3002/webhooks/payments
```
