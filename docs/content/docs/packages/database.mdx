bacon aims to provide a robust, type-safe database client that makes it easy to work with your data while maintaining strong typing throughout your application. We aim to support tooling that:

- Provide type-safe database client
- Handle real-time data synchronization
- Offer powerful query capabilities with full TypeScript support

## Active Edition: Convex

The active database adapter is [Convex](https://convex.dev) — a serverless, real-time database with no migrations required. This is the recommended and default configuration.

## Usage

Database configuration is located in `@repo/database`. You can use Convex queries and mutations from any server component:

```tsx title="page.tsx {1,4}"
import { api } from '@repo/database/convex/_generated/api';
import { fetchQuery } from 'convex/nextjs';

const Page = async () => {
  const pages = await fetchQuery(api.pages.listAll);

  // Do something with pages!
}
```

## Schema Definition

The Convex schema is defined in `packages/database/convex/schema.ts`. This file uses Convex's schema definition syntax to describe your database tables, relationships, and types.

Auth tables are managed by the `@convex-dev/better-auth` component and defined in the component's schema. You should not modify auth tables directly.

### Adding a new table

Add your table to `packages/database/convex/schema.ts`:

```ts title="packages/database/convex/schema.ts"
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  // ... existing tables
  posts: defineTable({
    title: v.string(),
    content: v.string(),
    authorId: v.id("users"),
  }).index("by_author", ["authorId"]),
});
```

### Deploying changes

```sh title="Terminal"
pnpm migrate
```

This runs `npx convex deploy` to push your schema changes to Convex.

## Other Adapters (Stubs)

bacon also includes stub configurations for [Prisma](/migrations/database/prisma-postgres) and [Drizzle](/migrations/database/drizzle) as alternate adapters. These are not active by default but can be switched by editing `packages/database/index.ts`:

```ts
// Convex edition (active)
export * from "./convex";

// Prisma edition (switch by uncommenting)
// export * from "./prisma";

// Drizzle edition (switch by uncommenting)
// export * from "./drizzle";
```

See [migrations](/migrations) for detailed migration guides from Convex to other databases.
