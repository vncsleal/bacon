# Project Context: bettercone-starter

## Stack (read this before writing any code)

| Layer | Technology | Key fact |
|-------|-----------|----------|
| Auth | Better Auth 1.x | NOT Clerk, NOT NextAuth |
| Database | Convex | NOT Prisma/SQL — no migrations, no DATABASE_URL |
| Payments | Stripe | via `@better-auth/stripe` plugin |
| Framework | Next.js App Router | Turborepo monorepo, pnpm, TypeScript strict |

---

## Where Things Live

```
packages/auth/
  better-auth/client.ts    # authClient — use in client components
  server.ts                # getToken, getSessionInfo — use in RSC/server actions
  middleware.ts            # authMiddleware — wraps Next.js middleware
  permissions.ts           # RBAC: owner/admin/member role definitions

packages/database/
  index.ts                 # ASSEMBLY ENGINE WRITES THIS — do not edit
  convex/
    auth.ts                # createAuth factory + Convex queries (getCurrentUser etc.)
    schema.ts              # App-level Convex tables (page, etc.)
    betterAuth/schema.ts   # Auth tables (user, session, org, member, apikey...)
    _generated/            # Regenerated by `npx convex dev` — do not edit

packages/payments/
  index.ts                 # ASSEMBLY ENGINE WRITES THIS — do not edit
  providers/stripe/        # Active Stripe integration

apps/app/
  app/api/auth/[...all]/   # Better Auth route handler (the auth server)
  app/(authenticated)/     # Protected routes — all require valid Convex JWT
  app/(better-auth)/       # sign-in, sign-up pages
  app/actions/             # Server actions (users/, organizations/)
  middleware.ts            # Applies authMiddleware + security headers
```

## Reading Auth in RSC

```ts
import { getToken } from "@convex-dev/better-auth/nextjs";
import { createAuth } from "@repo/database/convex/auth";
import { api } from "@repo/database/convex/_generated/api";
import { fetchQuery } from "convex/nextjs";

export default async function Page() {
  const token = await getToken(createAuth);
  if (!token) redirect("/sign-in");
  const user = await fetchQuery(api.auth.getCurrentUser, {}, { token });
  // ...
}
```

## Querying Convex in RSC

```ts
const data = await fetchQuery(api.myModule.myQuery, { arg }, { token });
```

## Using Auth in Client Components

```ts
"use client";
import { authClient } from "@repo/auth/client";

const { data: session } = authClient.useSession();
const { data: org } = authClient.useActiveOrganization();
```

## Do Not

- Add Prisma: `packages/database/prisma/` is a stub, not active
- Add Drizzle: `packages/database/drizzle/` is a stub, not active  
- Use `process.env` directly — use the typed `env` from `@/env`
- Edit `packages/database/index.ts` — assembly engine owns it
- Import `createAuth` in client components — server-only
- Reference Clerk or NextAuth — neither exists in this project

---

## Adding Features

**New data table:**
1. `packages/database/convex/schema.ts` → add `defineTable(...)`
2. `packages/database/convex/myTable.ts` → add queries/mutations
3. `cd packages/database && npx convex dev` → regenerates `_generated/`

**New org permission:**
1. `packages/auth/permissions.ts` → add resource + per-role access
2. `packages/database/convex/auth.ts` → already imports `ac, owner, admin, member`

**New payment plan:**
1. `packages/database/convex/auth.ts` → add to `stripe.subscription.plans[]`
2. Create the price in Stripe dashboard, set env var
